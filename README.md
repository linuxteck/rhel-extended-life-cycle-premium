## ⚡ 1-Minute Understanding

If you're short on time:

- ELS = Extended support after RHEL end-of-life  
- Security patches continue (critical fixes only)  
- Designed for legacy systems that can't upgrade immediately  

💡 It buys you time — not a permanent solution.

Many enterprises still run older RHEL versions in production.

- Understand RHEL lifecycle phases  
- Know when ELS is required

---

## 🔄 RHEL Lifecycle (Simplified)

| Phase | Duration | Support Level |
|------|---------|--------------|
| Full Support | ~5 years | Features + bug fixes + security |
| Maintenance Support | ~5 years | Security + critical fixes |
| Extended Life Cycle (ELS) | Optional | Critical security fixes only |

## 🚀 When Should You Use ELS?

```bash
# You cannot upgrade due to legacy apps
# Your system is still business-critical
# Compliance requires security patches
# Migration is planned but not completed
```

---

## ⚠️ When NOT to Use ELS

```bash
# For long-term strategy (not recommended)
# For new deployments
# When upgrade is easily possible
```
